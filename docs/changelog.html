Changelog

All notable changes to dck runtime, dck panel, and dck-wings.

dck Runtime

v1.12.0 — June 2026

  • New commands: cp, top, system prune, rename, commit, info
  • cp — copy files between host and containers (container:path syntax)
  • top — list running processes inside a container
  • prune — remove all stopped containers
  • rename — rename existing containers
  • commit — create images from container snapshots
  • info — display system and storage information
  • Improved error messages and logging

v1.11.0 — May 2026

  • Container command execution (dck exec)
  • Log viewing (dck logs)
  • Image management (pull, images, rmi)
  • OverlayFS storage layer
  • Environment variable support in create
  • Port mapping (--port flag)
  • Volume mounting (--volume flag)
  • Restart policies (--restart always|no)
  • Resource limits: memory (--memory) and CPU (--cpus)
  • Drone sub-command for game server presets (Minecraft, Rust, SRCDS, etc.)

v1.10.0 — April 2026

  • Initial public release
  • Core container lifecycle: create, start, stop, restart, rm, ps
  • Namespace isolation (PID, Network, Mount, UTS)
  • Cgroup resource management
  • Seccomp security profiles
  • OverlayFS for efficient storage
  • Bridge networking
  • Socket collection for running containers

dck Panel

v1.12.0 — June 2026

  • dck-wings integration — optional HTTP API mode for container/image operations
  • ClientInterface abstraction with local CLI and wings HTTP backends
  • New env vars: DECK_WINGS_URL, DECK_WINGS_API_KEY
  • Backend refactored for interface-based dispatch

v1.11.0 — May 2026

  • Image management UI (list, pull, remove)
  • Container presets / blueprints
  • Admin panel with user management
  • Settings page with configurable port range
  • Container stats display
  • SSH/SFTP server for container file access
  • Improved error handling across all handlers
  • System stats endpoint with cgroup/meminfo parsing
  • Auto-update mechanism (git pull + rebuild)
  • Install script with full automation

v1.10.0 — April 2026

  • Initial release
  • Container CRUD (create, start, stop, restart, delete)
  • WebSocket console with xterm.js
  • File manager (browse, read, write, upload, delete)
  • Backup system (create/restore/delete tar.gz archives)
  • JWT authentication
  • User roles (admin/user)
  • Dashboard with system stats
  • Dark glassmorphism UI

dck-wings

v1.0.0 — June 2026

  • Initial release
  • REST API for container operations (list, create, start, stop, restart, remove, logs, exec, stats)
  • WebSocket console terminal
  • File manager API (list, read, write, upload, delete, mkdir, rename)
  • Image API (list, pull, remove)
  • Bulk operations: prune, stop-all
  • Bearer token and query parameter authentication
  • TOML configuration file
  • Systemd service support
  • Linux builds: amd64 + arm64
