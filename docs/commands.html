Commands

Complete reference for the dck command-line interface.

Container Management

dck create

Create a new container from an image.

dck create <image> [--name name] [--port host:container] [--volume host:container]
         [--env KEY=VAL] [--restart always|no] [--memory 512M] [--cpus 2]
         [--network host|bridge] [--detach|--no-detach] [-- cmd arg1 arg2]
FlagDescription
--nameContainer name
--portPort mapping (host:container)
--volumeVolume mount
--envEnvironment variable
--restartRestart policy: always | no (default: no)
--memoryMemory limit (e.g. 512M, 2G)
--cpusCPU cores limit
--networkNetwork mode: host | bridge
--detach / --no-detachRun in background (default: --detach)

dck ps

List containers.

dck ps [--all] [--quiet]
FlagDescription
--allShow stopped containers too
--quietShow only container IDs

dck start | stop | restart

Control container lifecycle.

dck start <id>
dck stop <id>
dck restart <id>

dck rm

Remove a container.

dck rm <id> [--force]

dck exec

Execute a command inside a running container.

dck exec <id> <command>

dck logs

View container logs.

dck logs <id>

dck rename

Rename a container.

dck rename <id> <new_name>

dck top

Show running processes inside a container.

dck top <id>

dck cp

Copy files between host and container.

dck cp <source> <dest>

Use id:path syntax for container paths: 

# Copy from container to host
dck cp mycontainer:/app/config.json ./config.json

# Copy from host to container
dck cp ./config.json mycontainer:/app/config.json

dck commit

Create a new image from a container's current state.

dck commit <id> [name[:tag]]

Image Management

dck images

List downloaded images.

dck images

dck pull

Download an image from a registry.

dck pull <image>[:tag]

dck rmi

Remove a downloaded image.

dck rmi <image>

System Commands

dck info

Show system information (dck version, storage paths, container counts, etc.).

dck info

dck prune

Remove all stopped containers to free up disk space.

dck prune

dck version

Show dck version.

dck version

Drone (Game Server Presets)

dck includes drone — a sub-command for quick game server deployment from pre-configured presets.

dck drone <game> [--name name] [--port port] [--version version]
GameDescription
minecraftMinecraft Java Edition server
rustRust dedicated server
srcdsSource Engine dedicated server
vrisingV Rising dedicated server
enshroudedEnshrouded dedicated server
soulmaskSoulmask dedicated server
terrariaTerraria dedicated server
# Create a Minecraft server
dck drone minecraft --name mc-server --port 25565

# Start it
dck start mc-server

# View logs
dck logs mc-server
