dck-wings

REST API agent for remote container management. Acts as an intermediary between the dck panel and the dck runtime, providing an HTTP API for all container operations.

Architecture: Panel → Wings (HTTP/REST) → dck (CLI) — all on the same machine. Wings provides secure, authenticated access to container management.

Quick Start

# Download and install
curl -fsSL https://github.com/animesao/dck-wings/releases/latest/download/dck-wings-linux-amd64 -o /usr/local/bin/dck-wings
chmod +x /usr/local/bin/dck-wings

# Generate API key and start
WINGS_KEY=$(openssl rand -hex 32)
dck-wings --api-key "$WINGS_KEY" --port 8080

Install as System Service

# Using the built-in installer
dck-wings --install --api-key "$(openssl rand -hex 32)"

# Start and enable
systemctl start dck-wings
systemctl enable dck-wings

Configuration

dck-wings reads config from /etc/dck-wings/config.toml or CLI flags:

FlagEnvDefaultDescription
--portPORT8080HTTP server port
--hostHOST127.0.0.1Bind address
--api-keyAPI_KEY—Authentication key (required)
--dck-binDCK_BINdckPath to dck binary
--dck-homeDCK_HOME~/.dckdck data directory

Integrate with dck Panel

Set these environment variables on the panel server:

# In /etc/dck-panel.env or systemd Environment=
DECK_WINGS_URL=http://localhost:8080
DECK_WINGS_API_KEY=your-generated-key

When these variables are set, the panel will route all container and image operations through the wings API. File operations, console, and backups continue to work through the local filesystem.

API Reference

Authentication

All requests require either an Authorization: Bearer <key> header or ?api_key=<key> query parameter.

Containers

MethodPathDescription
GET/api/containersList containers (add ?all=1 for stopped)
POST/api/containersCreate a container
GET/api/containers/:id/stateGet container state
POST/api/containers/:id/startStart container
POST/api/containers/:id/stopStop container
POST/api/containers/:id/restartRestart container
POST/api/containers/:id/removeRemove container (?force=1)
GET/api/containers/:id/logsGet container logs
POST/api/containers/:id/execExecute command
GET/api/containers/:id/statsResource usage stats
GET/api/containers/:id/consoleWebSocket console terminal

Files

MethodPathDescription
GET/api/containers/:id/files?path=...List directory
GET/api/containers/:id/files/read?path=...Read file content
POST/api/containers/:id/files/writeWrite file
DELETE/api/containers/:id/files?path=...Delete file/directory
POST/api/containers/:id/files/mkdirCreate directory
POST/api/containers/:id/files/renameRename file/directory
POST/api/containers/:id/files/uploadUpload file (multipart)

Images

MethodPathDescription
GET/api/imagesList images
POST/api/imagesPull image
DELETE/api/images/:nameRemove image

System

MethodPathDescription
POST/api/system/pruneRemove all stopped containers
POST/api/system/stop-allStop all running containers
